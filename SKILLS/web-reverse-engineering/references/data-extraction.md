# Data Extraction Patterns

Extraction quality determines downstream model and analytics quality.

## Extraction Priority Order

1. native JSON API response
2. embedded structured data (JSON-LD/microdata)
3. stable DOM selectors (CSS/XPath)
4. LLM-assisted extraction for messy pages

## Structured Data First

Many pages expose product/article metadata in JSON-LD.

```python
from bs4 import BeautifulSoup
import json

soup = BeautifulSoup(html, "html.parser")
node = soup.find("script", {"type": "application/ld+json"})
if node and node.string:
    data = json.loads(node.string)
```

Use this before brittle CSS selectors whenever available.

## CSS vs XPath

| Method | Strength | Weakness |
|---|---|---|
| CSS selectors | concise and readable | weaker for text-relative navigation |
| XPath | strong structural navigation and text predicates | harder to maintain/read |

## Robust Selector Rules

- avoid dynamic class names generated by build pipelines
- anchor on semantic attributes (`data-*`, aria labels, stable ids)
- add fallback selectors for known variants
- validate extracted fields with schema constraints

## Pagination and Incremental Crawls

Capture:
- next-page token or cursor
- total count if provided
- stable sorting key for resume

Store crawl checkpoints so interrupted jobs resume deterministically.

## LLM-Assisted Extraction (Selective)

Use when pages are highly variable and structured hints are absent.

Guardrails:
- keep deterministic schema contracts
- run validator pass after LLM extraction
- keep raw HTML snapshot for replay and audit

## Normalization Contract

Persist at least three layers:

1. `raw_record` (source payload/snippet)
2. `normalized_record` (clean fields)
3. `extraction_meta` (url, timestamp, parser_version, confidence)

This enables reprocessing when parser logic changes.

## Common Failure Modes

| Failure | Fix |
|---|---|
| null spikes after layout change | add selector fallback set and smoke tests |
| duplicated items | deduplicate on canonical key (url/id/hash) |
| encoding artifacts | normalize unicode/whitespace early |

High-quality extraction pipelines are versioned data products, not one-off scripts.
