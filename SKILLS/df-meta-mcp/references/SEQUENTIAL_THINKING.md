# SEQUENTIAL_THINKING

This is the advanced helper family.

Use it only when the task genuinely needs **explicit step-by-step planning, branch/revision tracking, or tool recommendation bookkeeping**.

Routine GitHub / TickTick / docs lookups do **not** need this.

## Good use cases

- planning a multi-step workflow before acting
- comparing multiple tool paths
- keeping track of revisions / branches in a long reasoning chain
- asking for structured tool recommendations and execution order

## Notes

- It expects an explicit list of available tool names.
- It is more of a planning/orchestration surface than a domain API.
